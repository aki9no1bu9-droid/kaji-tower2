# フレンズタワー 🏆

あらかじめ用意した友達の画像がランダムに出てきて、積み上げる2P交互プレイのバランスゲーム。
落とした方が負け。スマホ（iPhone / Android）のブラウザで遊べます。

## ファイル構成

```
friends-tower/
├── index.html      画面の構造（HTML）
├── style.css       見た目（CSS）
├── README.md       これ
├── gen_placeholder.py  仮キャラ画像を作ったスクリプト（参考用）
├── images/         ★ ここに友達の透過PNGを入れる
│   ├── friend1.png （今は仮キャラ）
│   └── ...
└── js/
    ├── photos.js   画像の読み込み＋輪郭から当たり判定を自動生成
    ├── game.js     物理演算とゲームのルール
    └── main.js     画面切替・ボタン操作（司令塔）
```

## 自分の画像に差し替える方法

1. 友達の写真を **背景透過** して PNG にする
   （remove.bg などの無料サイトでOK）
2. その PNG を `images/` フォルダに入れる
3. `js/photos.js` の先頭にある `IMAGE_FILES` のリストを、自分のファイル名に書き換える

```js
const IMAGE_FILES = [
  'images/tanaka.png',
  'images/sato.png',
  // 好きなだけ追加
];
```

枚数は何枚でもOK。ゲーム中ランダムに出てきます。

## しくみ（ハイブリッド方式）

- **見た目** … 透過PNGをそのまま表示（友達の形のまま）
- **当たり判定** … PNGの不透明部分を読み取って、ざっくりした多角形を自動生成

輪郭そのままの細かい判定はスマホで重くなるので、
「見た目はキレイ・物理は軽い」のいいとこ取りにしてあります。
（動物タワーバトルも同じ考え方）

## VS Code での動かし方

1. このフォルダを VS Code で開く
2. 拡張機能「**Live Server**」をインストール
3. `index.html` を右クリック →「Open with Live Server」

※ HTMLを直接ダブルクリックで開くと、画像の読み込みがブロックされて
　 動かないことがあります。必ず Live Server 経由で開いてください。

## 当たり判定の精度を変えたいとき

`js/photos.js` の `buildOutline` 内：
- `STEPS`（既定16）… 大きくすると輪郭が細かくなる（重くなる）
- `> 40`（alphaしきい値）… 透明と判定する基準
